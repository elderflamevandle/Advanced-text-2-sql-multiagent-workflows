Implement the full `agents/nodes/executor.py`:

1. **Safety check**: Import `scan_sql` from `database.safety`. Call it on `state["generated_sql"]`. If not safe, return structured error_log dict and call `audit_blocked_query`. Set `error_log` to the structured dict: `{"error_type": "blocked_query", "message": friendly message per CONTEXT, "dialect": db_type, "failed_sql": sql, "hint": "Rephrase as a SELECT query."}`.

2. **LIMIT injection**: Before execution, check if the SQL has a LIMIT clause (case-insensitive regex `\bLIMIT\s+\d+`). If not, append `\nLIMIT 1000`. Be careful not to inject LIMIT inside subqueries -- only append to the outermost query. Simple approach: check if the last statement-level token area has LIMIT; if not, append. Do NOT inject LIMIT if the query is a CTE (WITH ... SELECT) that already has LIMIT in its final SELECT.

3. **Execution with timeout**: Use `concurrent.futures.ThreadPoolExecutor` with a single worker and `future.result(timeout=timeout_seconds)`. Load timeout from config.yaml (default 60). On `TimeoutError`, return `{"error_type": "timeout", "message": "{dialect} error: Query exceeded {timeout}s timeout", "dialect": db_type, "failed_sql": sql, "hint": "Try adding LIMIT or simplifying the query."}`.

4. **Normal execution**: Call `db_manager.execute_query(sql)`. On success, capture `execution_time_ms` (via `time.monotonic()` before/after) and `row_count` (len of results). Return `{"db_results": results, "error_log": None, "execution_metadata": {"execution_time_ms": elapsed_ms, "row_count": len(results)}}`.

5. **Error handling**: Wrap execute_query in try/except. On any exception, build structured error dict: `{"error_type": classify_error(exc), "message": "{dialect} error: {str(exc)}", "dialect": db_type, "failed_sql": sql, "hint": generate_hint(exc)}`. Use a simple `_classify_error(exc)` helper that maps exception types to error_type strings (e.g., `OperationalError` -> "syntax_error", `ProgrammingError` -> "column_not_found", generic Exception -> "unknown").

6. **Null checks**: If `generated_sql` is None/empty, return error_log with `error_type="missing_sql"`. If `db_manager` is None, return error_log with `error_type="no_connection"`.
