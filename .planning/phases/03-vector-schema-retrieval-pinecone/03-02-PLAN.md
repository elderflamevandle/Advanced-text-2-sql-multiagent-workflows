2. Create `vector/retriever.py` with:

   a. `BaseRetriever` ABC:
      - Import from `abc` (ABC, abstractmethod)
      - Three abstract methods:
        - `embed_schema(self, schema: dict[str, Any], namespace: str) -> None`
        - `retrieve_tables(self, query: str, namespace: str, top_k: int = 5) -> dict` — returns dict with keys: `tables` (list of table name strings), `table_metadata` (list of dicts with column info), `join_hints` (list of JOIN hint dicts from SchemaGraph)
        - `namespace_exists(self, namespace: str) -> bool`

   b. `PineconeRetriever(BaseRetriever)`:
      - `__init__`: lazy import `from pinecone import Pinecone, ServerlessSpec`. Read config from `config/pinecone_config.yaml` using `yaml.safe_load`. Initialize `self._pc = Pinecone(api_key=os.environ["PINECONE_API_KEY"])`. Call `self._ensure_index()`. Set `self._index = self._pc.Index(config["index_name"])`. Create `self._embedder = EmbeddingGenerator()`.
      - `_ensure_index()`: check `self._pc.has_index(self._config["index_name"])`. If not, create with `ServerlessSpec(cloud=config["cloud"], region=config["region"])`, dimension and metric from config.
      - `namespace_exists(namespace)`: call `self._index.describe_index_stats()`, check if `f"{namespace}:tables"` is in `stats["namespaces"]`.
      - `embed_schema(schema, namespace)`:
        - If `self.namespace_exists(namespace)`: log info "Namespace {namespace} already exists, skipping embedding" and return early
        - Build table texts using `build_table_text()` for each table
        - Build column texts using `build_column_text()` for each column in each table, extracting sample values from sample_rows
        - Embed all table texts via `self._embedder.embed_documents(table_texts)`
        - Embed all column texts via `self._embedder.embed_documents(column_texts)`
        - Upsert table vectors to `f"{namespace}:tables"` namespace with IDs `f"{namespace}::{table_name}"` and metadata `{"table_name": table_name, "text": table_text}`
        - Upsert column vectors to `f"{namespace}:columns"` namespace with IDs `f"{namespace}::{table_name}::{col_name}"` and metadata `{"table_name": table_name, "column_name": col_name, "data_type": col_type, "sample_values": str(sample_vals[:3])}`
        - Log progress during embedding
      - `retrieve_tables(query, namespace, top_k=5)`:
        - Get query embedding: `list(self._embedder.embed_query_cached(query))`
        - **Stage 1 — table-level**: query `f"{namespace}:tables"` with `top_k=top_k`, `include_metadata=True`
        - Extract `matched_table_names` from results metadata
        - **Stage 2 — column-level**: query `f"{namespace}:columns"` with `top_k=top_k * 10`, `include_metadata=True`, `filter={"table_name": {"$in": matched_table_names}}`
        - Build `SchemaGraph` from the FULL schema (need schema passed in — add `self._schema_cache: dict | None = None` set during `embed_schema`, or accept schema as param). Actually, retrieve_tables needs the schema to build SchemaGraph. Two options: (a) cache schema during embed_schema, (b) pass schema to retrieve_tables. Option (a) is cleaner — store `self._schema_cache = schema` in embed_schema. If schema_cache is None (e.g., schema was embedded in a prior session), retrieve_tables cannot do FK expansion — log warning and skip FK expansion.
        - FK expand: `graph = SchemaGraph(self._schema_cache)`, `expanded = graph.expand_tables(matched_table_names)`, `join_hints = graph.generate_join_hints(expanded)`
        - Return `{"tables": expanded, "table_metadata": column_metadata_list, "join_hints": join_hints, "scores": score_dict}`

   c. `ChromaRetriever(BaseRetriever)`:
      - `__init__`: lazy import `import chromadb`. Create `chroma_path = Path.home() / ".text2sql" / "chroma"`, `chroma_path.mkdir(parents=True, exist_ok=True)`. Initialize `self._client = chromadb.PersistentClient(path=str(chroma_path))`. Create `self._embedder = EmbeddingGenerator()`. Set `self._schema_cache = None`.
      - `_collection_name(namespace, tier)`: return `f"{namespace.replace(':', '_')}_{tier}"` — sanitize colons to underscores for ChromaDB collection name validity.
      - `namespace_exists(namespace)`: check if `self._collection_name(namespace, "tables")` exists in `[c.name for c in self._client.list_collections()]`.
      - `embed_schema(schema, namespace)`: same logic as PineconeRetriever but using ChromaDB `get_or_create_collection` and `upsert(ids=..., embeddings=..., metadatas=...)`. Store `self._schema_cache = schema`.
      - `retrieve_tables(query, namespace, top_k=5)`: get query embedding, query tables collection, extract matched table names, query columns collection with `where={"table_name": {"$in": matched_table_names}}`, do FK expansion via SchemaGraph if schema_cache available.

   d. `get_retriever() -> BaseRetriever` factory function:
      - `import os`
      - If `os.getenv("PINECONE_API_KEY")`: return `PineconeRetriever()`
      - Else: log warning "PINECONE_API_KEY not set - using ChromaDB local fallback", return `ChromaRetriever()`
